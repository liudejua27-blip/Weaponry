| --- | --- | --- |
| 文档分层和删除路线 | `DOCUMENTATION_MAP.md`、docs walkthrough | 本轮验证后更新 |
| Codex 执行合同 | `AGENTS.md`、`CODEX_*` | 计划证据，不是运行时能力 |
| 产品验收故事 | `PRODUCT_DEFINITION.md` | 目标；以任务和 Gate 退出 |
| GitHub 参考/Skill | `AGENT_GITHUB_REFERENCE_ARCHITECTURE.md`、`AGENT_PLUGINS_SKILLS_DESIGN.md` | 参考，不是依赖或功能 |

## 5. 维护规则

1. README、用户指南或 UI 新增能力时，必须更新本表并提供当前 Gate；
2. Gate 失败必须写 `KNOWN FAIL`，不得继续列为自动化通过证据；
3. “部分实现”必须列明未支持子能力；
4. 真实 Provider、签名、公证和外部服务测试只有显式执行后才能记录 PASS；
5. 文档整理后运行 docs walkthrough、repository integrity、safety、secrets 和 `git diff --check`。
