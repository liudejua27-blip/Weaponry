# ADR-0006：武器优先产品范围修订

- 状态：Accepted
- 日期：2026-07-10
- 决策者：项目维护者
- 取代：ADR-0001 第 6 项

## 背景

ADR-0001 将首版范围限定为低风险单零件，并拒绝武器及武器零部件。项目维护者随后明确：ForgeCAD 新产品不拒绝武器设计；前期产品设计应主要围绕武器展开，因为武器外形、组件、尺寸、角度、接口、材料和制造约束更直观，也更适合验证精细参数化工作流。

## 决策

1. ForgeCAD 支持武器、武器零部件和相关精密机械结构，不按类别自动拒绝。
2. 前期产品演示、工作台内容、固定回归样本和第一个 CAD 纵向切片以模块化武器项目为主。
3. 新武器设计仍使用通用 `DesignSpec`、`FeatureGraph`、Design/Version/Build/DFM 领域；不恢复旧 `WeaponDesignSpec`、CreativeWeaponGraph、SkillGraph、神经 3D 或 Unity 主链。
4. 涉及爆压、热载荷、疲劳、材料批次、安全关键接口或法定合规时，系统允许设计，但必须显式记录未验证假设和所需的专业验证，不得自动声称安全、合规或已认证。
5. L 型支架继续作为 CAD 内核 calibration fixture，用于快速几何和 round-trip 回归，但不再是首个产品里程碑。

## 后果

- README、设计、计划、操作文档、UI 和质量样本必须使用一致的武器优先边界。
- 旧 `release:safety-scope` 中“禁止现实武器制造语义”的断言只作为 legacy baseline 证据，不得进入 ForgeCAD C01–C10 新发布门。
- R2 合同必须能表达模块化组件接口、锁定参数、风险假设和未验证条件。
- R3 工期和测试矩阵需要覆盖多组件参数与接口回归，不能只证明一个简单支架能够构建。

## 被否决方案

- 继续在文档中拒绝武器、但在 UI 中展示武器：产品范围自相矛盾。
- 恢复旧 Weapon 领域并在其上增加 STEP：旧美术资产语义仍不能成为 CAD 工程源。
- 将工程警告实现成类别拒绝：无法满足维护者明确的产品定位。
