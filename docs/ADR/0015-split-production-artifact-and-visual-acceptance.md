# ADR-0015：拆分生产概念工件管线与生产级视觉验收

- 状态：Accepted
- 日期：2026-07-16
- 决策者：项目维护者
- 取代：ADR-0014 中“完整 M108 退出后才开始 K001”的依赖结论
- 补充：ADR-0010 的单一最佳结果、ADR-0011 的 3D 机械设计系统、ADR-0014 的 Rust-first 迁移

## 背景

原 `FGC-M108` 同时承担两种不同性质的退出条件：

1. 可由代码独立验证的生产概念工件管线：多区 PBR、GLB/readback、质量、正式导出、缓存、工作台加载和资源预算；
2. 依赖完整组件语言与真人评审的生产级视觉基线：四领域比例、材质可读性和表面细节分别达到独立评审中位数 `4/5`。

当前真实工作台截图证明，双档 GLB、512×512 PBR 和确定性缓存可以完成，但固定 showcase 仍主要由绝对坐标 primitive、Loft 和 Sweep 拼装。真正的 child slot、connector、局部变换、语义比例、材质区和可复用部件外观属于 `FGC-C105 EditableComponentRecipe@1`。

若继续要求完整 M108 先通过人工视觉门，C105 永远不能开始；若直接降低视觉门，又会把高分辨率低模错误描述为生产级概念资产。这是任务依赖环，不是单纯缺少评审者。

## 决策

1. 原 `FGC-M108` 标记为 `superseded`，历史检查点和脚本名称保留，不删除证据。
2. 新增 `FGC-M108A`：生产概念工件真值与轻量 PBR 管线。
   - 同一不可变 ShapeProgram 派生 `interactive_preview` 和 `production_concept`；
   - preview 服务编辑反馈，production 服务质量、展示、正式导出和下载；
   - 两档都必须有代码所有的 profile hash、GLB extras 和真实 readback；
   - production 使用按需内容寻址缓存和二进制传输，不创建第二资产版本链；
   - 退出只能说明“生产概念工件管线已验证”，不能说明模型视觉达到 `4/5`。
3. 新增 `FGC-M108B`：Recipe 驱动的生产级概念视觉基线。
   - 决策时必须等待 K003 和 C105；K003 已于 2026-07-18 满足，当前仍等待 C105；
   - 四领域固定基准必须由 `EditableComponentRecipe@1` 实例化，而不是把 M108 showcase 的手写绝对坐标视觉件当最终证明；
   - 每领域至少三份 recipe-backed production fixture 通过 M108A/Q003/G826 自动门；
   - 至少三位与实现独立的真人在同一 ForgeCAD `production_concept` 视口评分；
   - 每个领域的 `proportion`、`material_readability`、`surface_detail` 中位数分别不少于 `4/5`；
   - 任一领域失败时继续迭代 Recipe/资产，不降低门槛、不用代理评分补齐。
4. 主链改为：

```text
A004
→ M108A
→ K001
→ K002
→ K003
→ C105
→ M108B
→ V003
→ F026
→ A005
→ R007
→ D006
```

5. `agent:m108-*` 和 `M108_VISUAL_BENCHMARK_PROTOCOL.md` 保留历史命名兼容：
   - 自动生产工件 Gate 归 M108A；
   - 独立人工视觉评分协议归 M108B。
6. 当前软件仍可称本机 Alpha；只有 M108B 通过后，用户指南才可把基准覆盖的四领域输出称为“生产级概念资产基线”。仍不得称为照片级保证、工程 CAD、制造级模型或对所有提示普遍保证。

## 后果

- K001 不再等待外部人工评分，只等待 M108A 的工件真值和回归 Gate。
- C105 在 K003 的 Rust-owned core 上实现，然后由 M108B 验证它是否真正提升完整产品外观。
- V003 只消费通过 M108B 的候选质量基线，再负责内部多候选评审和唯一 winner；M108B 不实现候选选择或 UI。
- 用户无需为 M108A、K001–K003 或 C105 提供资料。M108B 最终退出时才需要组织三位独立评审；授权参考图和材质是可选增强，不是依赖解锁条件。

## 被否决方案

- 继续在 M108 固定 showcase 中无限堆 primitive：只能优化四个样例，不能形成 Agent 可复用的组件设计语言。
- 删除或降低 `4/5` 门槛：会把管线通过冒充视觉达标。
- 在 K003 前实现 C105 的第二套 Python 状态真值：会增加 Rust/Python 迁移双写风险。
