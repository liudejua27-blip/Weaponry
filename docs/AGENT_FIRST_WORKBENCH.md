# Agent 优先工作台

## 目标

ForgeCAD 面向零基础用户时，默认流程固定为：

1. 描述想法；
2. 选择一个 Agent 设计方向；
3. 点击部件进行简单调整；
4. 在底部确认保留或撤销；
5. 按用途导出。

CAD 的 ModuleGraph、Connector、坐标系、原始资产 ID、ChangeSet 和文件格式不会出现在首次操作路径中。它们仍由后端校验、记录和版本化。

## 界面规则

- 顶栏只保留项目名、自动保存状态、撤销、检查、导出。
- 左栏是唯一的 Agent 工作区：自然语言输入、概念家族和三条方向卡。
- 中央始终是单一 Three.js 视口。
- 选中部件时只出现上下文编辑卡：替换、让 Agent 调整、隐藏、大小、位置、方向。`精确调整` 默认折叠。
- 组件库仅在点击“替换”后从底部出现；首次只展示真实兼容的三个以内候选，用户可主动点“查看更多”。
- 每次 Agent 或手动修改都先进入底部确认条；只有“保留此修改”会创建新版本。
- 导出用使用目的组织，而不是首先要求用户理解 GLB、OBJ、PNG 或 ZIP。

## Agent 交互边界

Agent 的上下文包括当前项目、已选部件、已注册模块和当前预览状态；动作只允许生成方向、提出受限修改、替换兼容组件、预览、确认或撤销。实现采用与 tldraw Agent Template 相同的“用户选择 + 当前视图 + 可确认动作”原则，但不引入其 2D 画布运行时，也不替换现有 Three.js/Tauri 视口。

所有产物限定为虚构、非功能性的游戏、影视与展示概念道具；不提供现实武器制造、性能或结构工程能力。

## 验证

`npm run desktop:r3-concept-workbench-smoke` 覆盖：

- 默认 Agent 工作区、无常驻属性面板；
- 用途导出抽屉；
- 上下文替换抽屉与真实兼容候选；
- 组件替换预览、底部确认和版本持久化；
- Agent 修改预览、撤销；
- 按需打开模型检查抽屉；
- Agent 重启后确认版本回读。
