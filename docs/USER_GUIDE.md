# ForgeCAD 零基础用户指南

版本：2026-07-15
状态：本机 Alpha 当前功能说明

ForgeCAD 当前是轻量通用机械概念 3D Agent 的本机 Alpha。代码已提供从创意、三方向、轻量 blockout 到 Agent 资产编辑和 GLB 的最小纵向切片；当前桌面工作台 smoke 已通过 Agent-first 路径，但仍是受限测试流程，不能视为生产级功能。

每次明确领域的 Agent Turn 现在会在服务端通过受限产品工具完成一个临时候选的构建、真实 GLB 回读、四视图渲染和硬门检查。该内部候选不会自动保存，仍须用户后续确认；当前桌面尚未完成“只显示一个最佳结果”，三方向文本卡仍会显示到 V003。

本文只描述当前代码路径和已知验证边界。系统可以在装配与几何事实足够时给出受限的“预览拆分/预览合并”建议，并支持锁定、隐藏或单独查看当前部件；自由拆分/合并、完整版本历史浏览、多格式 Agent 资产导出和工程 CAD 能力不属于当前用户功能；本轮 Gate 状态以 [能力—Gate 矩阵](evidence/CAPABILITY_GATE_MATRIX.md) 为准。

## 1. 当前适合做什么

首批支持的概念领域：

- 未来武器概念道具；
- 汽车与地面载具；
- 飞机与航空器；
- 机械臂与机器人机构。

当前结果是非功能性的概念数字资产，不是制造模型、结构设计、车辆安全分析、适航分析或机器人控制方案。新生成的 ShapeProgram GLB 已包含内置视觉 PBR 纹理和固定工作室展示环境，用于改善材质可读性；这不等于照片级真实、真实材料或工程性能保证。

## 2. 第一次启动

当前建议在项目目录运行本机 Tauri 测试版：

```bash
script/build_and_run.sh --verify
```

该命令会构建桌面壳、启动本地 Agent 并检查健康状态。它依赖开发机现有的 Python 环境，不是可以直接发送给其他用户的独立安装包。

首次进入工作台后，先点击“创建第一个设计”，等待项目和 `ActiveDesignSnapshot` 初始化，再在设计助手顶部打开“配置模型服务”：

1. 填写 API Base URL；
2. 填写模型名称；
3. 填写 API Key；
4. 选择“保存并连接”；
5. 等待界面确认 metadata、Keychain、Agent 重启和本地 capability 均已就绪；
6. 使用“测试连接（会联网）”执行一次真实 Provider 检查。

macOS Tauri 运行时把 API Key 保存到系统 Keychain。仅写入 metadata 或 Keychain 不代表已经连接；只有新 Agent 进程的 capability 为 `ready` 时，界面才允许真实连接测试。测试或普通模型请求期间可以选择“取消本次模型请求”。取消、鉴权失败、余额不足、限流、服务错误、超时、空内容、非法 JSON 或 Schema 不符都不会创建计划、资产版本或导出，也不会静默改用离线结果冒充成功。不要把密钥写进项目文件、终端命令、截图或聊天记录。浏览器开发预览不提供 Keychain，开发者应按 [开发与调试](DEVELOPMENT.md) 使用本机 secret file。

即使没有配置大模型，当前工作台仍可使用确定性离线 Planner 验证基本流程；离线结果不能代表真实模型质量。

## 3. 创建概念

在 Agent 输入框中明确写出对象类别、完整外观和视觉要求。当前领域识别仍基于有限关键词，建议直接使用“汽车”“飞机”“机械臂”或“未来武器概念道具”等词。

推荐示例：

```text
设计一辆双座冰原探索汽车，完整封闭车身，大轮胎，短前后悬，深色耐候外观。
```

```text
设计一架紧凑的垂直起降概念飞机，宽机身、双短翼、深色舱罩，用于科幻救援展示。
```

```text
设计一台三关节维护机械臂，固定基座、两段连杆、旋转腕部和夹持末端。
```

```text
设计一个厚重、紧凑、非功能性的未来武器概念道具，用于游戏展示。
```

如果描述同时出现多个类别，或没有包含可识别类别词，Agent 会先保存一条澄清记录并只问一个普通问题。选择“汽车与地面载具”“飞机与航空器”“机械臂与机器人机构”或“未来武器概念道具”后，Agent 会保留原始创意并开启新一轮规划；在选择前不会创建 3D 模型、版本或资产。

## 4. 选择方向并生成预览

确定性离线 Planner 当前返回三个结构化方向。每个方向包含标题、简要外形、主要部件角色和材质方向；这不是四领域真实 Provider 质量评测结果。

方向卡当前显示结构化文字；工作台不再在 Turn 完成后自动并发请求三张方向概念图。服务端 Action Loop 已为其中一个临时候选执行真实 GLB readback 和四视图检查，但当前界面只把这些步骤作为 Tool Item 生命周期事实显示，不把内部候选冒充已保存模型或最终最佳结果。

选择一个方向后，主视图生成轻量 blockout：

- 每张方向卡会由系统从同一类别的预审视觉外观中稳定选取一版，例如不同的座舱、翼面、机械臂连杆布局或未来概念道具外壳；不需要输入参数，也不会显示技术编号；
- 如果外观不符合创意，可在“保存为可编辑模型”前点击“换一版外观”。系统只会在当前方向的三种预审视觉外观中换到下一版，并显示“当前第 N / 3 版”；这只替换未保存预览，不影响任何已保存设计、版本或导出；
- 当前方向流程只使用受限的轻量 ShapeProgram；后端已通过门禁的 primitive 包括 box/cylinder/wedge/capsule，以及 profile/extrude/revolve/loft/sweep/mirror/array/radial_array、受限 union/subtract 和 bevel_approx/surface_panel。union/subtract 只处理已验证封闭输入，由单一 Manifold 内核执行并在 GLB 中保留可回读特征历史；零基础用户界面不提供自由轮廓、放样、扫掠、布尔、倒角或面板编辑；
- 主视图会完整显示当前方向已使用的 box、cylinder、wedge 和 capsule 外形，并使用同一视口的柔化展示边缘、阴影和工作室光照帮助观察轮廓；这些只是显示层效果，不会改变模型几何、版本或材料事实；
- 在输入框下可选择“快速草图”或“展示模型”。快速草图用于先看整体轮廓；展示模型（默认）会在同一概念模型上增加有限的外观面板、分缝视觉线、护板、孔洞/紧固件点缀、灯带和线缆槽视觉线，并用石墨、复合外观、金属外观和发光色提高层次。它们只是外观沟通语言，不表示真实的开孔、散热、电气、固定、材料或功能结构。切换档位只会重建尚未保存的预览；已经保存的设计不会被自动修改；
- 用于验证整体比例、主要部件和编辑流程；
- 两种档位都不代表真实材料、功能结构、工程细节或照片级最终美术；
- 选择方向不会覆盖已经确认的资产版本。

在“下载当前设计”抽屉中，当前 Agent 设计只显示三个直接动作：点击“下载 3D 模型 (GLB)”取得当前概念级展示模型；点击“生成概念图”生成四张只读概念图；生成后可点击缩略图下载单张 PNG，或点击“下载概念图包”取得当前 PNG 与来源、尺寸和哈希清单。具备可安全一一对应部件几何的资产还会生成一张透明背景的“爆炸概念图”；它只帮助观察已有部件层级，不是装配、维修或制造说明。所有概念图结果绑定当前 `AgentAssetVersion`；若模型版本或概念图已更新，软件会要求先重新生成，避免下载不一致的图片。概念图包不包含模型源文件、工程图或制造资料；概念图只是外观沟通用的软件栅格化预览，不是照片级渲染、工程图或制造依据。

在已经保存为 Agent 资产后，主视图底部可以切换四个相机视图（等轴、正视、顶视、右视）和三个灯光预设（CAD 中性、柔和棚拍、概念对比）。这些选择会保存到当前 `ActiveDesignSnapshot`，重新打开同一项目会恢复；它们只改变同一个主视口的观察方式，不代表工程照明、照片级渲染或多视图图片导出。

## 5. 保存为可编辑资产

预览满意后选择“保存为 Agent 资产候选”。系统会创建一个独立的 `AgentAssetVersion`（初始阶段为 `segmented_concept`）并列出当前候选部件；首次使用旧 Concept 设计时，必须先显式选择“让 Agent 重建可编辑资产”，外部导入 GLB 仍是只读参考，不能直接提交。

当前可用的部件操作：

- 选择一个候选部件；
- 对出现“可调参数”的部件，按已声明的范围和步长减少或增加比例；界面会显示当前值、范围和每次调整量。没有该区域的部件暂不支持单独调整，软件不会猜测参数；
- 对出现“外观比例配方”的部件，可选择“更紧凑”“更修长”“更厚重”或“更简洁”等领域化按钮。配方只在当前部件已有受限比例参数且真实 GLB 回读能证明其表面来源时出现；点击后仍先形成临时预览，取消不写版本，确认才创建子版本。界面中的“倍”是相对外观比例，不是毫米、结构、性能或制造建议；
- 对支持的机械臂关节应用固定角度预览；
- 应用目录中已有的视觉材质；
- 把当前部件保存为项目内可复用组件；
- 用同角色、兼容组件进行替换。
- 锁定当前部件，避免 Agent 对该部件创建修改预览；
- 临时隐藏当前部件，或选择“只看这个部件”聚焦观察；需要时选择“显示所有部件”恢复。

部件列表会使用中文名称，例如“机身”“主翼”“座舱罩”“肩部关节”或“肘部关节”。这是对内部角色的显示翻译，不改变模型、版本或适配判断；遇到尚未登记的角色会显示“未命名部件”，不会猜测它的用途。

项目内可复用部件会先显示“来源检查通过”或“来源检查有提示”，再允许替换。可替换的最低条件是：同一领域、同一部件类型、组件处于启用状态，且其来源模型已有可用的最新检查结果；替换会保留当前部件在装配中的连接位置。来源未检查或检查失败时不会出现替换动作，需先检查来源模型。该提示只说明当前概念资产的替换边界，不代表工程结构、安全、材料或制造适配，也不等同于正式资产审阅。

每次修改先形成临时预览。只有选择“保留并创建新版本”才会写入新的 Agent 资产版本；选择取消不会写入该修改。

确认的 Agent 修改可以使用顶栏“撤销 / 重做”。每次撤销或重做都会创建一个新的可恢复版本，不会覆盖原来的历史版本；如果当前还有“临时预览”，请先保留或取消它，再执行撤销或重做。

当选中部件出现“预览拆分”或“预览合并”时，说明 Agent 已从当前装配关系和受限几何中找到了明确的候选。点击后仍只产生临时预览；确认才会创建新版本。没有建议时，表示当前事实不足，软件不会猜测切割线、连接方式或工程结构。

锁定、隐藏和单独查看会保存到当前工作台状态：锁定会让该部件的比例、材质、替换和结构建议动作不可用；隐藏或单独查看只改变当前主视图，不会删除部件、不会生成新版本，也不是工程装配锁定。若隐藏或单独查看使当前部件不可见，系统会取消该部件的选择。它们会随同一项目恢复，但版本切换后只保留仍存在的部件状态。

当前不支持自由输入任意尺寸、自由拆分/合并几何、任意版本列表跳转或精确运动学编辑。

## 6. 使用视觉材质

当前内置 13 个、覆盖六类的轻量视觉材质预设，例如石墨深灰、拉丝铝、汽车漆、橡胶和深色玻璃。新生成的工作台 GLB 会在真实 UV/tangent 和稳定材质区上嵌入内置的 base color、metallic-roughness、normal、occlusion、emissive 五通道视觉纹理；透明外观和涂层只在兼容时使用受限 glTF 扩展。打开外观抽屉后，先看当前部件/材质区；如果该部件有多个材质区，可先选择区域，再用名称/标签搜索或中文分类筛选。每张卡片显示用户登记纹理对象状态和来源摘要，点击“预览材质”只会生成当前部件/区域的临时 ChangeSet；确认后才创建新版本。没有可用的用户登记纹理时会显示“使用参数外观”，不会把它伪装成用户纹理已加载。材质只改变显示效果，不代表真实材料牌号、强度、重量或制造工艺。

材质修改也需要经过预览和确认。外部 GLB 参考模型的原始材质不会自动转换为可编辑 Material Zone。

## 7. 导入 GLB 参考

工作台可以导入满足限制的自包含 glTF 2.0 GLB：

- 检查文件结构、访问器、三角形预算和内容哈希；
- 将原始文件保存到内容寻址对象库；
- 在同一个主视图中显示；
- 允许原样导出。

导入的 GLB 当前是只读参考，不会自动重建为 ShapeProgram，也不会自动深度分件。它也不会自动进入 Agent Turn 上下文；需要编辑时，请另行用文字描述重新生成轻量模型，参考驱动重建属于目标能力。

## 8. 检查和导出

对当前 Snapshot 绑定的 Agent 资产，代码路径提供轻量质量检查：

- ShapeProgram 和 GLB 是否可读取；
- 同一次受限编译后的 GLB 三角形数量和边界是否有效；
- 装配层级和部件引用是否有效；
- 是否存在概念级 AABB 重叠预警。

这些检查不包含精确网格碰撞、结构强度、空气动力学、车辆安全或机器人动力学。
编译或 GLB 回读失败时，界面应显示检查不可用，不会用旧估算数字代替。

当前 Agent 资产正式支持的用户导出目标是 GLB；另外支持从当前资产生成四视图概念 PNG，以及条件满足时的透明背景爆炸概念 PNG。Agent 下载抽屉不会显示 OBJ、MP4 或源包；这些仅属于旧 Concept 的只读兼容路径，不应作为新 Agent 资产能力使用。PNG 和图包不会创建版本，也不改变质量、选择或 Snapshot。

导出前只检查工作台当前 `ActiveDesignSnapshot` 显示的 Agent 资产版本。Agent 路径的版本、选择、质量和导出已由同一 Snapshot 绑定；如果界面报告版本冲突或 `legacy_concept_read_only`，停止导出并记录问题，不要手工切换版本号。

旧项目需要核对历史信息时，可在旧版提示中点击“查看旧版只读信息”。该入口只显示 Graph Inspector、旧参数、旧质量摘要和曾支持的旧格式说明，不能编辑、检查或新建旧格式导出；关闭入口、切换项目或重新进入 Agent 资产后，这些旧版细节不会进入 Agent 的选择、质量或导出流程。

## 9. 保存与恢复边界

后端会保存已确认的 Agent 资产版本、部件、材质绑定和组件记录。桌面端会尝试恢复当前项目对应的 Agent 资产头。

服务端 S008/C104 路径已让 Agent 资产恢复、分件选择、部件显示/锁定、检查、撤销/重做和 GLB 导出读取同一 `ActiveDesignSnapshot`；当前桌面 Agent-first smoke 已通过并验证浏览器重启恢复，但这不等于 packaged 安装包的全新机器恢复已经完成。旧 Concept 兼容界面为只读重建路径，不要把两类版本号混在一起理解。重要测试前先执行备份；恢复操作见 [故障恢复手册](DISASTER_RECOVERY.md)。

## 10. 已知限制

- 在同一项目的同一设计会话中，你可以继续用普通语言说“把座舱压低一些”“让外观更简洁”这类修改；软件会保留近期对话、已确定的对象类别和当前可编辑设计摘要。若要从汽车改为飞机或机械臂，请新建设计会话，避免把两种设计混在一起；
- 智能生成会自动控制思考深度和本机日额度。达到当日额度时，软件会停止新的联网生成，但不会影响已保存设计的查看、编辑、检查或导出；

- 含糊的对象类别会显示一个选择问题；选择前不会创建模型、版本或资产。明确涉及现实制造、性能、安全、飞行/控制或认证的描述会在本机提示你改为“非功能外观概念”，不会发送给模型，也不会创建 3D 模型、版本或导出；这是一组有限的产品范围规则，不是完整内容安全判断；
- 当前方向流程仍主要展示 Planner 返回的三个方向；后端已具备受控 wedge/capsule/profile/extrude/revolve/loft/sweep/mirror/array/radial_array、单一 Manifold 内核的受限 union/subtract、bevel_approx、surface_panel，以及四领域各 12 个结构不同的 blockout 变体（G807）。零基础界面只提供当前方向三项族的“换一版外观”预览轮换，不开放完整变体目录、编号或自由参数；这些结果仍是概念级低多边形外观，不是工程 CAD；
- 你可以用“紧凑/延展/工业感、简洁/细节丰富、深色/信号色、低姿态/抬升展示”等普通外观词帮助 Agent 选择已有概念外观族；系统只在四个领域各自的预审外观中选择，不会把这些描述解释成尺寸、加工、性能或自由建模命令。未识别的外观词会安全回退到默认方向；
