# ForgeCAD 零基础用户指南

版本：2026-07-20
状态：本机 Alpha 当前功能说明

ForgeCAD 当前软件仍是轻量通用机械概念 3D Agent 的本机 Alpha。F026 已把旧三方向 UI 收敛为 Codex 式单结果工作台；V003 现在会让 Rust Agent 每个 Turn 只完成一次完整合成，并且只在真实编译/readback、四视图同源、Brief、比例、部件 role、五通道 PBR、Recipe 可编辑性和来源 Gate 全部通过后显示一个正式结果。交互反馈继续使用 `interactive_preview`；当前机械臂黄金路径的正式结果、质量、展示、下载和导出使用 1024×1024 五通道 PBR 的 `production_concept`。这不等于软件已经正式发布，也不等于模型外观已通过 M108B 独立真人 `4/5`。

每次明确领域的 Agent Turn 现在会在服务端通过受限产品工具完成一个候选的构建、真实 GLB 回读、四视图渲染和 13 项硬门检查。失败时不会展示或保存模型；只有两个可确定性修复的表面问题能在同一设计意图下最多原位修复两次。通过后界面只显示一个未保存结果，用户选择“确认保存”才创建资产版本；“换一个思路”应开启新 Turn，而不是让用户在三个方向中筛选。

对机械臂领域，C106–C108 已提供一条可验证的黄金路径。Agent 会根据当前 Style Token 从 3 个已审阅风格配方中内部确定性选择 1 个，每 Turn 仍只合成一个模型。当前 service-display 基准会生成底座、回转台、三个关节护罩、两段连杆装甲、线缆束、夹爪和表面 trim 共 10 个可回读部件/9 个连接；packaged `production_concept` 为 101,248 triangles、120 primitives 和 1024×1024 五通道 PBR。A005 可在确认 V1 后创建带流线修饰的不可变 V2，并经 Snapshot/CAS、导出和重启恢复。当前一次完整双编译流程仍较慢，画面也未达到目标参考图；这是特定机械臂基准的工程闭环，不是所有提示的外观质量保证。

C107/M109A 已把当前源码中的 service-display 工件分成同源双档：交互预览为 18,324 triangles、128×128 五通道 PBR，按需展示/导出为 99,092 triangles、109 primitives、8 套材质和 1024×1024 五通道 PBR。表面抽屉使用 SVG 预览受限流线、图案和粗糙度/法线/发光意图；确认后，Rust 密封的 `SurfaceLayerProgram@1` 才会由受限 Python 编译器绑定到真实 Material Zone，五通道贴图及 hash 从最终 GLB 回读。部件选择、两点测量和剖切使用同一个 3D renderer。真实截图仍未达到目标图级装甲嵌合、关节机械层级和微表面细节，不能把 99k/1K 视为视觉验收。

新版 F026 首次打开会显示 Agent-first 空工作区，不再自动恢复旧版 localStorage 项目选择；已有旧项目和数据库不会被删除，仍可从左侧显式打开。

当前原生桌面的会话、Turn、步骤 Item、取消、模型服务检查、13 项受限产品工具，以及项目、版本、Snapshot、材质修改、质量和导出均由 Rust app-server/core 统一拥有；Python 只执行受限几何编译，不读取 Provider Key，也不能写数据库、对象库或 Snapshot。未配置模型服务时会明确显示未配置且不发起网络请求，失败或取消不会保存隐藏推理。Rust-first 核心所有权已经验证，但软件仍是本机 Alpha，不等于已签名发布或模型视觉已经通过 M108B。

本文只描述当前代码路径和已知验证边界。系统可以在装配与几何事实足够时给出受限的“预览拆分/预览合并”建议，并支持锁定、隐藏或单独查看当前部件；自由拆分/合并、完整版本历史浏览、多格式 Agent 资产导出和工程 CAD 能力不属于当前用户功能；本轮 Gate 状态以 [能力—Gate 矩阵](evidence/CAPABILITY_GATE_MATRIX.md) 为准。

## 1. 当前适合做什么

首批支持的概念领域：

- 未来武器概念道具；
- 汽车与地面载具；
- 飞机与航空器；
- 机械臂与机器人机构。

当前结果是非功能性的生产概念工件，不是制造模型、结构设计、车辆安全分析、适航分析或机器人控制方案。新生成的正式 ShapeProgram GLB 已包含 512×512 v4 内置视觉 PBR 纹理、稳定材质区和固定工作室展示环境；这只描述工件分辨率、readback 和导出真值，不等于照片级真实、真实材料、工程性能或已经达到生产级概念资产视觉基线。

## 2. 第一次启动

当前建议在项目目录运行本机 Tauri 测试版：

```bash
script/build_and_run.sh --verify
```

该命令会构建桌面壳、启动本地 Agent 并检查健康状态。它依赖开发机现有的 Python 环境，不是可以直接发送给其他用户的独立安装包。

首次进入工作台后，先点击“创建第一个设计”，等待项目和 `ActiveDesignSnapshot` 初始化，再在设计助手顶部打开“配置模型服务”：

1. 填写 API Base URL；DeepSeek 官方当前 OpenAI 格式地址为 `https://api.deepseek.com`；
2. 填写当前模型名称 `deepseek-v4-pro`（质量优先）或 `deepseek-v4-flash`（速度优先）；不要再为新配置使用即将弃用的 `deepseek-chat` / `deepseek-reasoner` 别名；
3. 填写 API Key；
4. 选择“保存并连接”；
5. 等待界面确认 metadata、Keychain、Agent 重启和本地 capability 均已就绪；
6. 使用“测试连接（会联网）”执行一次真实 Provider 检查。

macOS Tauri 运行时把 API Key 保存到系统 Keychain。普通打开工作台只读取不含密钥的 Provider metadata，Keychain 读取次数为 0，也不应要求输入 macOS 登录密码；用户主动点击一次“测试连接”时只读取一个短生命周期凭据快照，一次真实 Turn 也只读取一个快照，并在 preflight、预算检查和后续工具请求间复用，Turn 成功、失败或取消后立即释放。当前 V4 Agent Turn 显式使用思考模式和 Agent 级 `max` 强度；界面只显示步骤与最终结果，不显示或保存隐藏推理。保存/清除配置仍会按需访问 Keychain。仅写入 metadata 或 Keychain 不代表已经连接；只有新 Agent 进程的 capability 为 `ready` 时，界面才允许真实连接测试。测试或普通模型请求期间可以选择“取消本次模型请求”。取消、鉴权失败、余额不足、限流、服务错误、超时、空内容、非法 JSON 或 Schema 不符都不会创建计划、资产版本或导出，也不会静默改用离线结果冒充成功。当前本机 Alpha 是 ad-hoc 重建包，二进制变化后 macOS 可能把它视为新的请求者并再次显示钥匙串授权；这是应用身份授权，不表示登录密码已被 ForgeCAD 修改。不要把密钥写进项目文件、终端命令、截图或聊天记录。浏览器开发预览不提供 Keychain，开发者应按 [开发与调试](DEVELOPMENT.md) 使用本机 secret file。

未配置大模型时仍可查看、编辑和导出已经保存的设计，但不能把 legacy 离线 Planner 的输出当作 V003 正式生成结果。仓库中的 `offline_deterministic` 只用于 Rust 合同和开发回归；它证明生命周期与 Gate，不代表面向用户的模型质量或真实联网生成。

## 3. 创建概念

在 Agent 输入框中明确写出对象类别、完整外观和视觉要求。当前领域识别仍基于有限关键词，建议直接使用“汽车”“飞机”“机械臂”或“未来武器概念道具”等词。

推荐示例：

```text
设计一辆双座冰原探索汽车，完整封闭车身，大轮胎，短前后悬，深色耐候外观。
```

```text
设计一架紧凑的垂直起降概念飞机，宽机身、双短翼、深色舱罩，用于科幻救援展示。
```

```text
设计一台三关节维护机械臂，固定基座、两段连杆、旋转腕部和夹持末端。
```

```text
设计一个厚重、紧凑、非功能性的未来武器概念道具，用于游戏展示。
```

如果描述同时出现多个类别，或没有包含可识别类别词，Agent 会先保存一条澄清记录并只问一个普通问题。选择“汽车与地面载具”“飞机与航空器”“机械臂与机器人机构”或“未来武器概念道具”后，Agent 会保留原始创意并开启新一轮规划；在选择前不会创建 3D 模型、版本或资产。

## 4. 查看唯一生成结果并确认保存

正式 V003 路径不会生成三个完整模型再评分，也不会显示方向卡、方向计数、内部 variant ID 或原始 Gate JSON。Rust Agent 会选择当前受限 Domain Pack、Style Token、Recipe 与几何语法，完成一次 `production_concept` 合成；只有硬门全部通过时，工作台才加载与决定记录同 hash/profile 的 binary GLB。

工作台会把计划、工具、检查和结果作为连续步骤显示，但不会公开隐藏推理。结果卡中的“确认保存”会绑定当前 project/turn/preview/hash 并原子创建一个版本；取消、项目切换、迟到响应、关闭预览或所有 Gate 失败都不会推进项目 head。离线确定性测试与真实 DeepSeek 结果都会携带 Rust 绑定的来源标记；离线通过只证明流程，不代表真实模型服务质量。

当前生成与兼容边界：

- 正式 V003 只产生一个通过结果；若外观不符合创意，请创建新的设计 Turn。该操作不覆盖任何已保存设计、版本或导出；
- 旧项目、开发浏览器 fixture 与显式兼容流程仍可能生成 `compatibility_result`。它没有 V003 formal provenance、不会产生 `SingleResultDecision@1`，也不能被描述成正式生成结果；
- 当前流程只使用受限的轻量 ShapeProgram；后端已通过门禁的 primitive 包括 box/cylinder/wedge/capsule，以及 profile/extrude/revolve/loft/sweep/mirror/array/radial_array、受限 union/subtract 和 bevel_approx/surface_panel。union/subtract 只处理已验证封闭输入，由单一 Manifold 内核执行并在 GLB 中保留可回读特征历史；零基础用户界面不提供自由轮廓、放样、扫掠、布尔、倒角或面板编辑；
- 主视图先使用同源 `interactive_preview` 提供编辑反馈，再在后台加载同一 ShapeProgram/版本的 `production_concept` 并在同一个视口原子替换。当前机械臂正式档实际嵌入 1024×1024 五通道视觉 PBR、阴影和固定工作室光照；项目切换或迟到结果不能覆盖当前资产，production 失败时保留 preview 并显示原因。两档不创建第二版本链；
- 在输入框下可选择“快速草图”或“展示模型”。快速草图用于先看整体轮廓；展示模型（默认）会在同一概念模型上增加有限的外观面板、分缝视觉线、护板、孔洞/紧固件点缀、灯带和线缆槽视觉线，并用石墨、复合外观、金属外观和发光色提高层次。它们只是外观沟通语言，不表示真实的开孔、散热、电气、固定、材料或功能结构。切换档位只会重建尚未保存的预览；已经保存的设计不会被自动修改；
- 当前代表性的未来游戏道具展示外观会使用分层硬表面壳体、倾斜握持外观、侧面流线、错列视觉通风、灯带、铝边框和深色玻璃端面；这些仍是固定预审外观与非功能展示语言，不是现实武器机构、制造设计或任意自由雕刻；
- 用于验证整体比例、主要部件和编辑流程；
- 两种展示档位都不代表真实材料、功能结构、工程细节、照片级最终美术或 M108B 真人视觉达标；
- 正式未保存结果和 legacy compatibility preview 都不会覆盖已经确认的资产版本；只有正式结果的确认操作会创建新版本。

在“下载当前设计”抽屉中，当前 Agent 设计只显示三个直接动作：点击“下载 3D 模型 (GLB)”取得当前版本的 `production_concept` GLB；点击“生成概念图”生成四张只读概念图；生成后可点击缩略图下载单张 PNG，或点击“下载概念图包”取得当前 PNG 与来源、尺寸和哈希清单。具备可安全一一对应部件几何的资产还会生成一张透明背景的“爆炸概念图”；它只帮助观察已有部件层级，不是装配、维修或制造说明。所有概念图结果绑定当前 `AgentAssetVersion`；若模型版本或概念图已更新，软件会要求先重新生成，避免下载不一致的图片。`production_concept` 是工件 profile 名称，不是照片级或真人视觉通过标签；概念图包也不包含模型源文件、工程图或制造资料。

在已经保存为 Agent 资产后，主视图底部可以切换四个相机视图（等轴、正视、顶视、右视）和三个灯光预设（CAD 中性、柔和棚拍、概念对比）。这些选择会保存到当前 `ActiveDesignSnapshot`，重新打开同一项目会恢复；它们只改变同一个主视口的观察方式，不代表工程照明、照片级渲染或多视图图片导出。

## 5. 保存为可编辑资产

正式单一结果满意后选择“确认保存”。系统会创建一个独立的 `AgentAssetVersion` 并列出当前候选部件；首次使用旧 Concept 设计时，必须先显式选择“让 Agent 重建可编辑资产”，外部导入 GLB 仍是只读参考，不能直接提交。明确标记的 legacy compatibility 流程可能仍显示旧“保存为 Agent 资产候选”文案，但它不是 V003 正式决定。

当前可用的部件操作：

- 选择一个候选部件；
- 对出现“可调参数”的部件，按已声明的范围和步长减少或增加比例；界面会显示当前值、范围和每次调整量。没有该区域的部件暂不支持单独调整，软件不会猜测参数；
- 对出现“外观比例配方”的部件，可选择“更紧凑”“更修长”“更厚重”或“更简洁”等领域化按钮。配方只在当前部件已有受限比例参数且真实 GLB 回读能证明其表面来源时出现；点击后仍先形成临时预览，取消不写版本，确认才创建子版本。界面中的“倍”是相对外观比例，不是毫米、结构、性能或制造建议；
- 对支持的机械臂关节应用固定角度预览；
- 应用目录中已有的视觉材质；
- 把当前部件保存为项目内可复用组件；
- 用同角色、兼容组件进行替换。
- 锁定当前部件，避免 Agent 对该部件创建修改预览；
- 临时隐藏当前部件，或选择“只看这个部件”聚焦观察；需要时选择“显示所有部件”恢复。

部件列表会使用中文名称，例如“机身”“主翼”“座舱罩”“肩部关节”或“肘部关节”。这是对内部角色的显示翻译，不改变模型、版本或适配判断；遇到尚未登记的角色会显示“未命名部件”，不会猜测它的用途。

项目内可复用部件会先显示“来源检查通过”或“来源检查有提示”，再允许替换。可替换的最低条件是：同一领域、同一部件类型、组件处于启用状态，且其来源模型已有可用的最新检查结果；替换会保留当前部件在装配中的连接位置。来源未检查或检查失败时不会出现替换动作，需先检查来源模型。该提示只说明当前概念资产的替换边界，不代表工程结构、安全、材料或制造适配，也不等同于正式资产审阅。

每次修改先形成临时预览。只有选择“保留并创建新版本”才会写入新的 Agent 资产版本；选择取消不会写入该修改。

确认的 Agent 修改可以使用顶栏“撤销 / 重做”。每次撤销或重做都会创建一个新的可恢复版本，不会覆盖原来的历史版本；如果当前还有“临时预览”，请先保留或取消它，再执行撤销或重做。

当选中部件出现“预览拆分”或“预览合并”时，说明 Agent 已从当前装配关系和受限几何中找到了明确的候选。点击后仍只产生临时预览；确认才会创建新版本。没有建议时，表示当前事实不足，软件不会猜测切割线、连接方式或工程结构。

锁定、隐藏和单独查看会保存到当前工作台状态：锁定会让该部件的比例、材质、替换和结构建议动作不可用；隐藏或单独查看只改变当前主视图，不会删除部件、不会生成新版本，也不是工程装配锁定。若隐藏或单独查看使当前部件不可见，系统会取消该部件的选择。它们会随同一项目恢复，但版本切换后只保留仍存在的部件状态。

当前不支持自由输入任意尺寸、自由拆分/合并几何、任意版本列表跳转或精确运动学编辑。

## 6. 使用视觉材质

当前内置 13 个、覆盖六类的轻量视觉材质预设，例如石墨深灰、拉丝铝、汽车漆、橡胶和深色玻璃；这些目录项当前映射到 8 套规范内置 PBR 外观，因此部分预设会共享同一纹理实现，不能把 13 项描述为 13 套完全不同的纹理资产。新生成的工作台 GLB 会在真实 UV/tangent 和稳定材质区上嵌入内置的 base color、metallic-roughness、normal、occlusion、emissive 五通道视觉纹理，主视图会优先加载这一份同源 GLB；透明外观和涂层只在兼容时使用受限 glTF 扩展。在左栏展开“换一个视觉材质”，界面会读取当前活动资产已提交的 `part + zone` binding 作为选中状态，并只把与当前领域兼容的材质放入快捷列表；如果该部件有多个材质区，可先选择区域，再用名称/标签搜索或中文分类筛选。内置项会标明“内置五通道 PBR”；用户登记或导入纹理仍单独显示来源和可用状态。

点击“预览材质”会为当前活动部件和稳定 Material Zone 创建临时 ChangeSet。服务端先验证材质与领域兼容，再临时编译二进制 PBR GLB；桌面只有在该 GLB 成功回到同一视口后才显示确认入口。预览失败会自动取消 ChangeSet 并恢复当前已提交模型，不会继续显示旧材质或允许用户在看不到真实结果时确认。确认后才创建新版本并重新加载该版本的同源 PBR GLB。用户登记纹理对象不可用时仍会明确显示安全回退，不会冒充已加载。材质只改变显示效果，不代表真实材料牌号、强度、重量或制造工艺。

材质修改也需要经过预览和确认。外部 GLB 参考模型的原始材质不会自动转换为可编辑 Material Zone。

### 6.1 添加表面图案、流线和雕刻感细节

在工作台底部输入框旁的“+”菜单，或选中部件后的外观入口，可以打开“表面细节”抽屉。首次使用时必须明确点击“启用外观细节能力”；ForgeCAD 不会在后台自动启用 Skill。启用后可为当前活动 Agent 资产中已选中的部件和稳定 Material Zone 选择受限的图案、流线、浅雕刻感或纹理变化，并调整有界强度和覆盖率。

点击预览后，服务端会先验证部件、材质区、基础材质、Skill 版本和当前资产头，再把 `SurfaceAdornmentProgram@1` 烘焙成临时五通道 PBR GLB，并在原来的唯一 3D 视口中显示。确认才创建不可变子版本；取消、预览失败、项目切换或迟到响应都会恢复已提交版本。确认后的程序、内容哈希、Part、Material Zone、Skill ID/版本和基础材质会进入资产与 GLB 回读证据，因此撤销、重做和重启不会依赖浏览器本地状态。

该能力只产生概念外观：所谓“雕刻”是通过受控 normal/roughness/base-color 等表面通道形成的视觉雕刻感，不是实体切削、制造深度、刀路、结构特征或可加工结论。当前也不接受任意脚本、外部纹理 URL 或自由贴图节点；未选中可编辑部件、Skill 未启用或后端不能完成真实 GLB 预览时会明确拒绝。

## 7. 添加参考图或 GLB

工作台底部“+”可打开“参考图 / GLB 引导重建”。当前接受 PNG/JPEG/WebP（不超过 16 MB）和自包含 glTF 2.0 GLB（不超过 32 MB）。保存前必须填写来源和使用权利/许可声明，并可声明缺失视角与可见线索。

参考证据会：

- 验证文件格式并记录内容哈希；
- 以只读身份进入 Rust 内容寻址对象库，不会替换当前模型；
- 记录可见轮廓/比例/材质区提示与不确定性，不声称恢复背面、内部结构、尺寸或功能；
- 当项目已有一个当前可编辑 ForgeCAD 资产时，可用已审阅 Recipe 生成一个受限重建预览；确认才创建新版本，取消不改变 head。

若项目尚无可编辑基准，工作台会保存证据，但不开放重建预览；初始生成将由 V003 单次合成路径接管。R007B 已用单图、多视图 contact sheet 和严格 GLB readback 三类证据验证：参考会形成不同的受限分析、Recipe/Material Zone/A005 计划与新 GLB，并可在同一 packaged 工作台、同一 renderer 中复现“只读参考→新结果”。这证明参考驱动的工程闭环已经接通，但 manifest 明确标记 `visual_fidelity_validated=false`；软件仍不能保证新模型达到参考图的外观质量或相似度。

## 8. 检查和导出

对当前 Snapshot 绑定的 Agent 资产，代码路径提供轻量质量检查：

- ShapeProgram 和 GLB 是否可读取；
- 同一次受限编译后的 GLB 三角形数量和边界是否有效；
- 装配层级和部件引用是否有效；
- 是否存在概念级 AABB 重叠预警。

这些检查不包含精确网格碰撞、结构强度、空气动力学、车辆安全或机器人动力学。
编译或 GLB 回读失败时，界面应显示检查不可用，不会用旧估算数字代替。

当前 Agent 资产正式支持的用户导出目标是 GLB；另外支持从当前资产生成四视图概念 PNG，以及条件满足时的透明背景爆炸概念 PNG。Agent 下载抽屉不会显示 OBJ、MP4 或源包；这些仅属于旧 Concept 的只读兼容路径，不应作为新 Agent 资产能力使用。PNG 和图包不会创建版本，也不改变质量、选择或 Snapshot。

导出前只检查工作台当前 `ActiveDesignSnapshot` 显示的 Agent 资产版本。Agent 路径的版本、选择、质量和导出已由同一 Snapshot 绑定；如果界面报告版本冲突或 `legacy_concept_read_only`，停止导出并记录问题，不要手工切换版本号。

旧项目需要核对历史信息时，可在旧版提示中点击“查看旧版只读信息”。该入口只显示 Graph Inspector、旧参数、旧质量摘要和曾支持的旧格式说明，不能编辑、检查或新建旧格式导出；关闭入口、切换项目或重新进入 Agent 资产后，这些旧版细节不会进入 Agent 的选择、质量或导出流程。

## 9. 保存与恢复边界

后端会保存已确认的 Agent 资产版本、部件、材质绑定和组件记录。桌面端会尝试恢复当前项目对应的 Agent 资产头。

服务端 S008/C104 路径已让 Agent 资产恢复、分件选择、部件显示/锁定、检查、撤销/重做和 GLB 导出读取同一 `ActiveDesignSnapshot`；当前桌面 Agent-first smoke 已通过并验证浏览器重启恢复，但这不等于 packaged 安装包的全新机器恢复已经完成。旧 Concept 兼容界面为只读重建路径，不要把两类版本号混在一起理解。重要测试前先执行备份；恢复操作见 [故障恢复手册](DISASTER_RECOVERY.md)。

## 10. 已知限制

- 在同一项目的同一设计会话中，你可以继续用普通语言说“把座舱压低一些”“让外观更简洁”这类修改；软件会保留近期对话、已确定的对象类别和当前可编辑设计摘要。若要从汽车改为飞机或机械臂，请新建设计会话，避免把两种设计混在一起；
- 智能生成会自动控制思考深度和本机日额度。达到当日额度时，软件会停止新的联网生成，但不会影响已保存设计的查看、编辑、检查或导出；

- 含糊的对象类别会显示一个选择问题；选择前不会创建模型、版本或资产。明确涉及现实制造、性能、安全、飞行/控制或认证的描述会在本机提示你改为“非功能外观概念”，不会发送给模型，也不会创建 3D 模型、版本或导出；这是一组有限的产品范围规则，不是完整内容安全判断；
- V003 已完成单次合成、13 项硬门、最多两次同意图原位修复和一个正式结果的确认链；C108 已完成机械臂 19,776-triangle preview、101,248-triangle/1K production、A005 V2、导出与重启恢复。R007B 已完成三类参考证据到同工作台可编辑结果的 exact-lineage 工程闭环，但没有验证视觉相似度；M108B 仍未完成四领域正式 kit 与独立真人 `4/5`，完整 M109 的四领域 2K 压缩 PBR 和设备分级也未完成；
