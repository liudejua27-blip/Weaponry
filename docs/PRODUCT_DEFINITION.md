# ForgeCAD 产品定义与范围

版本：2026-07-14
状态：P0 产品定义；当前实现是本机 Alpha，已完成 G1–G7 最小切片但未达到 P0 退出条件。

## 1. 产品一句话

ForgeCAD 是面向零基础用户的本地优先通用机械概念 3D Agent。用户只需要描述创意，Agent 负责理解、在内部生成和评审候选、选择一个最佳完整外观、生成轻量可编辑模型、拆分部件、应用视觉材质、检查并导出。

产品名称不再使用 “Weapon Concept Agent” 作为总产品名。武器只保留为第一批领域包之一：`future_weapon_prop`。

## 2. 首批领域包与通用范围

| 领域包 | 面向用户的说法 | P0 结果 |
| --- | --- | --- |
| `future_weapon_prop` | 未来武器概念道具 | 虚构游戏资产、影视道具、展示模型 |
| `vehicle_concept` | 汽车与地面载具 | 完整车身、座舱、轮组/履带和外观件 |
| `aircraft_concept` | 飞机与航空器 | 机身、机翼、尾部、舱罩和舱体外观 |
| `robotic_arm_concept` | 机械臂与机器人机构 | 基座、关节、连杆、腕部和末端工具 |

用户不进入四个 Mode，也不需要手动选择领域包。Agent 从描述中判断，无法判断时只问一个普通语言问题。当前服务使用有限关键词/同义词词表，并在 `ambiguous` 或 `unsupported` 时于 Planner 前停止，只持久化一个等待澄清的 Turn/Item，不写入 Plan、Blockout、版本或资产，也不会默认回退未来武器包。工作台提供四个面向零基础用户的选项，并在选择后保留原始 Brief 开启新一轮规划。

四个包是当前 Alpha 的首批验证范围，不是产品永久上限。目标系统通过同一 Core、版本化 Domain Pack 和产品 Skill 继续支持家用电器、工具设备、工程机械、农业机械、服务设备和其他生活机械的**非功能性完整外观概念**。新类别必须先有角色、比例、组件、材质、失败样例和质量 Gate，不能用一个“通用机械”默认包猜测所有对象，也不能据此提供制造、安全或认证结论。

## 3. 零基础用户主流程

以下是 P0 目标流程。当前可验证的缩减流程以 [USER_GUIDE.md](USER_GUIDE.md) 为准。

```text
说出创意
  → Agent 回述理解并在内部生成多个受限候选
  → 编译、GLB 回读、概念渲染并评审候选
  → Agent 只展示一个有证据的最佳完整外观
  → Agent 建议分件，用户接受或调整
  → 选中部件后用自然语言或简单编辑卡微调
  → 预览并确认变更，生成新的资产版本
  → 检查、渲染、导出
```

首屏只保留：Agent 对话、单一 3D 主视图、步骤状态、撤销、检查和导出。Connector、Joint、ShapeProgram、拓扑、坐标和格式等专业信息按需展开。

### P0 产品验收故事

方案只有同时满足以下故事，才算真正覆盖“通用机械概念 3D Agent”，而不是把 Weapon 模板换名字：

1. 用户分别输入未来武器道具、汽车、飞机和机械臂 Brief，Agent 都能在内部生成候选并只展示一个通过真实编译/readback 的**完整外观**结果；
2. 最佳结果是完整、比例可信且可解释的视觉 blockout，而不是单个枪形或局部零件；用户不需要在三张方向卡中替 Agent 评审；
3. 模型能转为可编辑资产，至少包含领域角色明确的层级分件；
4. 选中任一可编辑部件后，可以用普通语言调整比例/位置/概念姿态或替换兼容部件；
5. 可以只给指定部件/材质区域应用视觉材质，其他部分保持不变；
6. 每次永久修改都先预览、再确认、再创建新版本；取消不产生永久副作用；
7. 检查、当前选择、版本和导出始终指向同一 `ActiveDesignSnapshot`；
8. 新用户不安装神经 3D、Blender、Unity、CUDA、模型权重或插件，也能完成首次 GLB 导出；
9. 未识别领域进入一个简短澄清问题，不得默认变成武器；
10. 四领域固定 truth set、工作台 E2E、重启恢复和本机安装测试全部通过。

当前实现只覆盖其中一部分，因此产品仍是 Alpha。退出条件和任务顺序见 [CODEX_EXECUTION_PLAN](CODEX_EXECUTION_PLAN.md) 与 [CODEX_TASK_INDEX](CODEX_TASK_INDEX.md)。

## 4. 编辑能力边界

目标状态下，每个已确认部件都应能单独：

- 让 Agent 调整外观或比例；
- 修改尺寸、位置、旋转和概念姿态；
- 替换为同角色、同领域且通过质量检查的组件；
- 只给该部件或材质区域换视觉材质；
- 隐藏、隔离、锁定、撤销和恢复；
- 保存为可复用组件。

所有永久修改都走 `ChangeSet preview → confirm → new AgentAssetVersion`，不会静默覆盖当前版本。

当前只实现固定比例/位置/部分关节姿态、13 个六类视觉材质、项目内组件保存和同角色替换，以及已确认 Agent 修改的服务端撤销/重做；隐藏、隔离、锁定、自由参数、任意版本历史浏览和深度分件尚未完成。

确认前后的检查会提示未声明装配关系的部件是否在概念包围盒中明显重叠。它是零基础用户的防呆提示，不是精确网格碰撞、间隙、公差或运动学结论。

用户也可以导入自包含 `.glb` 作为**参考模型**。导入时应用验证 glTF 2.0、内嵌缓冲、外部资源、三角预算、访问器范围和 SHA-256；参考模型可以在同一主视图中查看、检查和原样导出，但不会被伪装成可编辑 ShapeProgram。要继续逐部件编辑，用户应让 Agent 依据参考模型重建一个新的可编辑概念资产。

## 5. 轻量技术路线

默认不安装本地神经 3D 模型、CUDA、ComfyUI 或模型权重。大模型 API 负责交流、规划和生成受限 `ShapeProgram`；本机 Geometry Worker 只执行白名单 primitive、分件、质量检查和 GLB 写出；桌面端始终复用单一 Three.js renderer。

P0 的目标是“完整、比例可信、外观接近真实产品、可编辑、低资源”。“接近真实产品”指曲面转折、部件接缝、视觉细节、Material Zone、PBR 纹理、透明/涂层表现、光照和色彩管理明显优于当前低多边形参数材质；它仍不等于工程精度、制造可行性或真实材料性能。DeepSeek 负责规划、评审与工具编排，本机受限几何和资产管线负责生成、验证和固化 3D。

## 6. 明确不做的事情

- 现实武器制造图纸、制造尺寸、机构、弹药、工艺或性能建议；
- 汽车安全、动力、碰撞或道路认证；
- 飞机空气动力学、结构强度、适航或飞行控制；
- 机械臂载荷、扭矩、碰撞安全、控制代码或工业认证；
- B-Rep/STEP 工程 CAD、完整 DFM、切片和制造就绪结论；
- 在 Agent 中执行任意 Python、JavaScript、shell 或外部 URL；
- 将旧任务中心、Mode、Forge、独立资产库重新变成产品导航。

## 7. 文档使用规则

文档中出现的能力必须标明状态：

- **已实现**：有代码路径和 smoke/E2E 证据；
- **兼容基线**：旧 Weapon/Unity/Provider 路径，只用于回归；
- **目标设计**：合同或界面已定义，但没有实现证据；
- **发布阻断**：需要真实 Provider 评测、外部 GLB 的自动重建/深度分件、碰撞/运动学、正式资产审阅或打包 sidecar。
